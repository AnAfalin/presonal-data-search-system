Система поиска персональных данных

Описание:

Разработать высоконагруженную систему поиска персональных данных. Система должна искать в отправленных на проверку файлах следующие данные:

  • Номера телефонов
  • Серия и номер паспорта
  • Дата рождения
  • E-mail
  • IP-адреса
  • Номера банковских карт

Бизнес-процесс:

Клиент отправляет файл на проверку в систему. Содержимое файла преобразуется в удобную форму для обработки и поиска ПД (персональных данных). На выходе клиент получает список найденных ПД или сообщение о том, что файл не содержит ПД.

Поддерживаемые форматы файлов:

  • Текстовые (docx, txt)
  • Табличные (xlsx, csv)
  • Архивы (zip, rar) – по желанию
  • Другие – по желанию

Архитектура:

Система должна быть спроектирована на основе микросервисной архитектуры.

Микросервисы системы:

Система должна включать как минимум следующие сервисы, которые должны общаться между собой по HTTP (REST) или по Kafka (способ общения выбрать в процессе разработки):

  • UI – графический интерфейс, позволяющий использовать API (с авторизацией)
  • AuthorizationServer – сервер авторизации
  • TaskService – сервис создания задачи на поиск
  • TextProcessor – сервис обработки текстовых файлов
  • TableProcessor – сервис обработки табличных файлов
  • ArchiveProcessor – сервис обработки архивов (по желанию)
  • CheckerService – сервис поиска КД.
  • В процессе разработки можно добавлять микросервисы по необходимости.

UI:

UI отвечает только за графический интерфейс (простой frontend). Все запросы, требующие логики, отправляются на backend.

Авторизация:

Авторизация должна выполняться только на UI-сервисе (с помощью AuthorizationServer).

Развертывание:

Система должна быть развернута с помощью Docker-Compose.
